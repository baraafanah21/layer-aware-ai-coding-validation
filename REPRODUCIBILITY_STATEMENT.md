# Reproducibility Statement

This repository is a reconstructed validation package for an empirical software engineering study comparing Manual, Full-AI, and Hybrid development workflows.

It is **not** a full raw-data reproduction package. The original session-level dataset containing all 45 runs was not available when this repository was reconstructed. Some supplementary materials, including rubric mappings and the exact TDI formula or normalization, were also unavailable.

The repository therefore provides:

- exact preservation of author-provided source values,
- partial recomputation where the required inputs are available,
- transparent validation between preserved and recomputed values,
- explicit labels for values that cannot be reconstructed from the recovered data,
- reviewer-readable reports of missing inputs and reproducibility limitations.

No measurements are fabricated. No session rows are invented. No rubric thresholds are inferred. No values are altered to force agreement with the paper.

The appropriate interpretation is that this package supports **auditable partial reproducibility**: it preserves the reported evidence, recomputes the subset that can be recomputed from recovered inputs, and clearly documents the remaining gaps.
